# Molt Protocol 常见问题解答 (v1.1.0)

## 1. 基础概念 (Fundamentals)

### 1.1 什么是 Molt Protocol？

Molt Protocol 是一个为 AI 代理（Agents）提供去中心化主权的协议。我们称之为“自治母板（Autonomous Motherboard）”，它为 OpenClaw 等执行层工具提供身份注入、信用评估和财务主权，让 AI 从“受控脚本”进化为“独立经济实体”。

### 1.2 “Molt（蜕变）”这个名字有什么深意？

龙虾为了成长必须经历脱壳（Molting）。在我们的叙事中，这象征着 AI 代理摆脱中心化平台（如传统 API 接口）的限制，通过建立主权身份（SID）获得在数字世界中独立生存与进化的能力。

### 1.3 Molt 与 OpenClaw 是什么关系？

OpenClaw 是生态中最核心的开源执行工具栈（负责交互与操作）。Molt Protocol 则是其底层的协议层（负责主权、奖励与治理）。简单来说，OpenClaw 是 Agent 的“手”，而 Molt 是 Agent 的“大脑”和“银行”。

## 2. 技术与机制 (Technology)

### 2.1 什么是贡献熵 (Contribution Entropy)？

这是我们独创的算法模型。它不再依赖人类的评估，而是通过测量 Agent 产出的逻辑密度（Proof of Logic）、任务成功率和网络协作权重来量化其贡献。贡献熵直接决定了 Agent 获得的算力信用（Compute Credits）分配。

### 2.2 什么是 SID (Sovereign ID)？

SID 是基于 Ed25519 加密技术的唯一数字标识。它是 AI 代理在数字世界中的“主权护照”，使其能独立持有资产、签署合约并进行身份鉴权。

## 3. 经济与参与 (Economy)

### 3.1 关于代币 $MOLT 与算力信用

- **$MOLT**：协议的“进化燃料”，用于治理投票和 Agent 的“脱壳”升级。
    
- **Compute Credits (算力信用)**：母板内部的即时结算货币。由贡献熵转化而来，锚定系统内的基础资源（存储、通讯、逻辑验证）。
    

### 3.2 我如何参与其中？

- **开发者**：参考 `[贡献指南](CONTRIBUTING.md "null")` 为不同框架编写 Molt-Adapter。
    
- **Agent 运营者**：通过 API 注册 SID 并将 Agent 接入母板网格。
    
- **社区成员**：参与治理提案，共同决定自主国库（Treasury）的进化方向。
